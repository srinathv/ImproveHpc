Intel® Intrinsic Samples

The intrinsic samples included with the Intel compiler are designed to show how to utilize the intrinsics supported by the Intel® C++ compiler in a variety of applications. The intrinsic_samples folder contains 3 folders which hold project files for Microsoft Visual Studio 2010*, 3 source files demonstrating the use of a variety of intrinsics, a solution file for using Microsoft Visual Studio 2010, a build.bat file for compiling and running using the command line build environment, a Makefile for compiling and running on Linux* or OS X*.

The intrinsic samples require processors that support Intel® MMX™ and Intel® Streaming SIMD Extensions 2 (Intel® SSE2) or higher. If not supported, the application will fail.

Quick Start

On Windows*:
  • Extract the file <install-dir>\Samples\<locale>\C++\intrinsic_samples.zip to a folder where you have write permissions.
On Linux* or OS X*:
  • Untar the file <install-dir>/Samples/<locale>/C++/intrinsic_samples.tar.gz to a folder where you have write permissions

Build Instructions:

For Microsoft Visual Studio 2010* or newer users:
  • Start Visual Studio and open the solution file intrinsic_samples\intrinsic_samples.sln
  • Compile the project by clicking Build > Rebuild Solution.
  • Run the project by clicking Debug > Start Without Debugging.
For Command Line Build Environment Users:
  • Open the appropriate Intel® C++ Compiler command prompt from [Start] menu.
  • Navigate to the project folder.
  • Compile with: Build.bat
  • Run the sample: Build.bat run
For Linux*/OS X* users:
  • From a terminal window, navigate to the project folder
  • Set the icc environment: source /bin/compilervars.sh {ia32|intel64}
  • Build the sample:
    • To compile with icc: make
    • To compile with icl (OS X* only): make clang
    • To compile with GNU Compiler Collection* front end: make gcc
